# Contributing

Contributions are welcome.

## Good first steps

1. Fork the repository.
2. Create a branch for your change.
3. Keep changes focused and easy to review.
4. Test the UI or batch scripts that your change touches.
5. Open a pull request describing what changed and why.

## Helpful contribution areas

- clearer documentation,
- accessibility and usability improvements,
- bug fixes,
- better install automation,
- tests around chunk splitting and stitching,
- and cross-platform improvements.

## Before sharing changes

- Do not commit generated audio, run logs, or local cache files.
- Do not commit private voice samples. The `samples` folder is intentionally configured to ignore local audio by default.
- Keep public documentation friendly to non-technical users when possible.
